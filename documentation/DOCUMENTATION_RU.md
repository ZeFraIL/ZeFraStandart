## 🎯 Цель проекта
•	**Какую задачу решает приложение:**  
Приложение является демонстрационным и учебным проектом, объединяющим в себе различные стандартные механизмы Android: работу с базами данных, мультимедиа (TTS/STT), сетевыми состояниями и сенсорами.

•	**Почему эта задача важна:**  
Она позволяет отработать взаимодействие между компонентами системы (Activity, Fragment, Service, BroadcastReceiver) в рамках одного приложения.

•	**Целевая аудитория:**  
Разработчики, изучающие стандартные возможности Android, и пользователи, нуждающиеся в удобном интерфейсе для ведения дневника, работы со ссылками и напоминаниями.

________________________________________
## 📌 Требования к приложению
**Функциональные требования**
•	Авторизация и регистрация пользователей.  
•	Отображение списка полезных ссылок с проверкой интернет-соединения.  
•	Голосовой ввод (Speech-to-Text) и воспроизведение текста (Text-to-Speech).  
•	Ведение личного дневника и создание напоминаний.  
•	Динамическое создание интерфейса в некоторых разделах.  

**Нефункциональные требования**
•	**Производительность:** Быстрый отклик за счет выноса тяжелых операций (БД, файлы) в фоновые потоки.  
•	**Удобство использования:** Интуитивная навигация через BottomNavigationView и NavigationDrawer.  
•	**Надёжность:** Предотвращение вылетов при отсутствии необходимых системных компонентов (телефон, почта).

## 🧠 Общая архитектура
•	**Выбранный подход:**  
MVC (Model-View-Controller).

•	**Почему выбран именно он:**  
Для данного масштаба проекта классическое разделение на контроллеры (Activity/Fragments) и модель (HelperDB/POJO) обеспечивает достаточную прозрачность без излишнего усложнения кода.

•	**Основные компоненты системы:**  
– `Activities` (Start, Welcome) — точки входа.  
– `Fragments` — модульные части интерфейса (Login, Feedback, Links).  
– `HelperDB` — управление данными SQLite.  
– `Receivers` — обработка системных событий.

## 🧩 Подробное описание классов
### 📌 Класс: Start
**Роль:** Сплеш-скрин (экран приветствия).  
**Ответственность:** Инициализация начальных настроек, приветствие пользователя через TTS.  
**Основные методы:**  
- `onCreate()` — настройка CountDownTimer для перехода на следующий экран.  

### 📌 Класс: BaseMenu
**Роль:** Утилитарный класс для управления меню.  
**Ответственность:** Централизованное управление PopupMenu и регистрация ресиверов (заряд батареи).  
**Основные методы:**  
- `showPopupMenu()` — отображение меню.  
- `handleMenuItemClick()` — логика кликов.  

### 📌 Класс: FeedbackFragment
**Роль:** Экран обратной связи.  
**Ответственность:** Запись аудио, преобразование речи в текст, сохранение в файл.  
**Основные методы:**  
- `startSpeechToText()` — активация голосового ввода.  
- `saveTextToFile()` — запись в локальное хранилище.

## 🔄 Диаграмма работы приложения
1. Пользователь запускает приложение (StartActivity).  
2. После таймера открывается WelcomeActivity с BottomNavigation.  
3. Пользователь выбирает раздел (например, "Ссылки").  
4. LinksFragment запрашивает данные у HelperDB.  
5. InternetConnectionReceiver проверяет доступность сети перед открытием ссылки.

## ⚙️ Работа с потоками
•	**Используются:**  
– `CountDownTimer` (StartActivity)  
– `AsyncTask` (GuideFragment — для чтения файлов)  
– `Thread` (в SignUp для работы с БД)  

•	**Почему выбран этот способ:** Простота реализации для разовых фоновых задач.  
•	**Предотвращение зависаний:** Тяжелые операции вынесены из главного потока (UI thread).
