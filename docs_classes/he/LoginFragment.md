# תיאור מפורט של המחלקה: LoginFragment

________________________________________
## 1. מידע כללי
•	**שם המחלקה:** LoginFragment  
•	**סוג:** Fragment (פרגמנט/רכיב מסך)  
•	**ייעוד:** פרגמנט זה אחראי על תהליך הכניסה של המשתמש למערכת. הוא בודק את שם המשתמש והסיסמה שהוזנו אל מול הרשומות בבסיס הנתונים המקומי.  
•	**אינטראקציה:** מוצג בתוך ה-Activity של `Welcome`. עם כניסה מוצלחת, הוא עובר ל-Activity של `Junction` ומעביר לשם את אובייקט ה-`User`.

________________________________________
## 2. משתנים (שדות המחלקה)
| שם | סוג | ייעוד | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `etNameL`, `etPassL` | `EditText` | שדות קלט לשם המשתמש והסיסמה. | ב-`onCreateView` לקבלת הנתונים שהמשתמש הזין. |
| `bLogin` | `Button` | כפתור אישור הכניסה. | ב-`onCreateView` להגדרת המאזין ללחיצה. |
| `helperDB` | `HelperDB` | מופע של עוזר בסיס הנתונים. | לצורך שאילתת טבלת המשתמשים. |
| `user` | `User` | אובייקט המייצג את המשתמש שנכנס. | מועבר למסך הבא לאחר כניסה מוצלחת. |

________________________________________
## 3. שיטות המחלקה
### שם השיטה: onCreateView
•	**סוג:** public  
•	**ערך חזרה:** View (תצוגת הפרגמנט)  
•	**פרמטרים:** `LayoutInflater`, `ViewGroup`, `Bundle`  
•	**מה השיטה עושה:**  
1. טוענת את עיצוב הפרגמנט מקובץ ה-XML.  
2. מקשרת את רכיבי הממשק למשתנים.  
3. מגדירה מאזין ללחיצה על כפתור ה-`bLogin`.  
4. בודקת שהשדות אינם ריקים.  
5. קוראת לשיטה `isUserFound()` כדי לאמת את הפרטים.  
6. אם המשתמש נמצא, היא יוצרת `Intent` למעבר ל-`Junction`, מצרפת את אובייקט ה-`User` ועוברת למסך הבא.

### שם השיטה: isUserFound
•	**סוג:** private  
•	**ערך חזרה:** boolean (true אם המשתמש נמצא, false אחרת)  
•	**פרמטרים:** `String stName`, `String stPassword`  
•	**מה השיטה עושה:**  
1. פותחת את בסיס הנתונים במצב קריאה.  
2. מבצעת שאילתה לטבלת ה-`Users`.  
3. עוברת בלולאה על כל הרשומות בתוצאה (`Cursor`).  
4. משווה את שם המשתמש והסיסמה שנשמרו לאלו שהוזנו.  
5. אם נמצאה התאמה, היא ממלאת את אובייקט ה-`user` ומחזירה `true`.

________________________________________
## 4. מחזור חיים
•	**onCreateView()**: נקודת הכניסה העיקרית של הפרגמנט. כאן מתבצעים אתחול רכיבי הממשק והגדרת המאזינים.

________________________________________
## 5. אינטראקציה עם הממשק (UI)
•	**EditText**: קריאת נתונים באמצעות `.getText().toString()`.  
•	**My_Toast**: משמש להצגת הודעות שגיאה באדום.  
•	**BaseMenuClass.speak()**: מספק משוב קולי לשדות ריקים או ניסיונות כניסה כושלים.

________________________________________
## 6. אינטראקציה עם רכיבים אחרים
•	**SQLite**: אינטראקציה ישירה עם `HelperDB` לשליפת נתוני משתמשים.  
•	**Intent**: משמש למעבר בין מסכים והעברת אובייקט מסוג `Serializable`.

________________________________________
## 7. לוגיקה כללית של המחלקה
המשתמש מזין שם וסיסמה. הפרגמנט בודק את רשומות בסיס הנתונים המקומי אחת אחת. אם נמצאה התאמה, המשתמש מאומת ומועבר למסך ה"צומת" (`Junction`). אחרת, מוצגת הודעת שגיאה המלווה בקריינות קולית.

________________________________________
## 8. הסבר מופשט
**הסבר במילים פשוטות:**  
חשבו על `LoginFragment` כעל **מאבטח בשער**. אתם מראים לו את תעודת הזהות שלכם (שם וסיסמה). הוא בודק בספר רישום גדול (בסיס הנתונים) כדי לראות אם אתם מופיעים ברשימה. אם אתם רשומים, הוא נותן לכם לעבור את השער לאזור הבא (`Junction`). אם לא, הוא אומר "הגישה נדחתה" ולא נותן לכם להיכנס.

**טיפ לשיפור:** כרגע, השיטה `isUserFound` עוברת על כל משתמש בטבלה. עבור כמות קטנה זה בסדר, אך עדיף להשתמש בפקודת SQL מסוג `WHERE` כדי למצוא את המשתמש הספציפי ישירות, דבר המהיר בהרבה ככל שבסיס הנתונים גדל.
