# תיאור מפורט של המחלקה: Start

________________________________________
## 1. מידע כללי
•	**שם המחלקה:** Start  
•	**סוג:** Activity (מסך אפליקציה)  
•	**ייעוד:** מחלקה זו אחראית על מסך הפתיחה (Splash Screen). היא מאתחלת את בסיס הנתונים, מגדירה את מנגנון המרת הטקסט לדיבור (TTS) ומעבירה את המשתמש באופן אוטומטי למסך הבא לאחר 4 שניות.  
•	**אינטראקציה:** מפעילה את ה-Activity בשם `Welcome` בסיום הטיימר או בלחיצה על כפתור. משתמשת ב-`HelperDB` להכנת בסיס הנתונים.

________________________________________
## 2. משתנים (שדות המחלקה)
| שם | סוג | ייעוד | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | שומר מידע על מצב האפליקציה הנוכחי. | באתחול בסיס הנתונים וה-TTS. |
| `ivLogo` | `ImageView` | רכיב להצגת הלוגו (תמונה). | בשיטה `onCreate` לצורך האנימציה. |
| `bStart` | `Button` | כפתור למעבר ידני למסך הבא. | בשיטה `onCreate` להגדרת המאזין ללחיצה. |
| `cdt` | `CountDownTimer` | טיימר לספירה לאחור. | לצורך מעבר אוטומטי לאחר 4 שניות. |
| `totalTTS` | `TextToSpeech` | משתנה סטטי להמרת טקסט לדיבור. | מאותחל כאן וניתן לשימוש במחלקות אחרות. |

________________________________________
## 3. שיטות המחלקה
### שם השיטה: onCreate
•	**סוג השיטה:** protected  
•	**ערך חזרה:** void (לא מחזירה דבר)  
•	**פרמטרים:**  
| שם | סוג | תיאור |
| :--- | :--- | :--- |
| `savedInstanceState` | `Bundle` | מצב המסך בעת שחזור. |
•	**מה השיטה עושה:**  
1. קובעת את עיצוב המסך מקובץ ה-XML (`activity_start`).  
2. מאתחלת את בסיס הנתונים דרך `HelperDB` כדי לוודא שהטבלאות נוצרו.  
3. מאתחלת את `totalTTS` כדי לאפשר השמעת טקסט.  
4. מקשרת את רכיבי הממשק (`ivLogo`, `bStart`) לקוד.  
5. מפעילה אנימציית סיבוב של הלוגו ב-360 מעלות.  
6. מגדירה מאזין ללחיצה על כפתור `bStart`, שמבטל את הטיימר ועובר למסך `Welcome`.  
7. מפעילה את הטיימר `cdt` למשך 4 שניות.  
•	**מתי נקראת:** באופן אוטומטי על ידי המערכת עם יצירת המסך.  
•	**חשוב להבין:** אם לוחצים על הכפתור, חובה לבטל את הטיימר (`cancel`) כדי למנוע מעבר כפול למסך הבא.

________________________________________
## 4. מחזור חיים (Activity/Fragment בלבד)
•	**onCreate()**: נקראת עם הפעלת האפליקציה. כאן מתבצעות כל הגדרות הרכיבים.

________________________________________
## 5. אינטראקציה עם הממשק (UI)
•	**ImageView (ivLogo)**: משמש לאנימציית סיבוב. מקושר באמצעות `findViewById`.  
•	**Button (bStart)**: מטפל בלחיצה (`OnClickListener`).

________________________________________
## 6. אינטראקציה עם רכיבים אחרים
•	**Intent**: משמש למעבר ל-`Welcome.class`.  
•	**SQLite**: קורא ל-`getWritableDatabase()` דרך `HelperDB`, מה שיוצר את הטבלאות אם הן לא קיימות.

________________________________________
## 7. לוגיקה כללית של המחלקה
עם הפעלת האפליקציה, המשתמש רואה לוגו מסתובב. בזמן זה, בסיס הנתונים מוכן ברקע. לאחר 4 שניות (או בלחיצה על כפתור), האפליקציה עוברת למסך הראשי.

________________________________________
## 8. הסבר מופשט
**הסבר במילים פשוטות:**  
תארו לעצמכם שזהו "חדר הכניסה" של הבית שלכם. בזמן שהאורח עומד כאן, אתם בודקים שהכל מוכן בחדרים (בסיס נתונים) ומפעילים את המוזיקה (דיבור). כדי שהאורח לא יחכה יותר מדי, אחרי 4 שניות אתם מלווים אותו בעצמכם לסלון (`Welcome`), או שהוא יכול להיכנס לשם מוקדם יותר על ידי לחיצה על כפתור "כניסה".

**טיפ לשיפור:** המשתנה `totalTTS` מוגדר כ-`static`. זה נוח, אך עלול לגרום לבעיות זיכרון אם לא סוגרים את ה-TTS בצורה נכונה בסיום השימוש.
