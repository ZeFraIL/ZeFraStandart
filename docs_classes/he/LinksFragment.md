# תיאור מפורט של המחלקה: LinksFragment

________________________________________
## 1. מידע כללי
•	**שם המחלקה:** LinksFragment  
•	**סוג:** Fragment (פרגמנט)  
•	**ייעוד:** מציג רשימה של קישורים שימושיים השמורים בבסיס הנתונים. מאפשר למשתמשים לצפות בסרטונים בנגן מובנה או לגלוש באתרים באמצעות WebView.  
•	**אינטראקציה:** טוען נתונים דרך `HelperDB`, משתמש ב-`InternetConnectionReceiver` לניטור חיבור הרשת, ומשתמש ב-`CustomAlertDialog` להצגת התוכן.

________________________________________
## 2. משתנים (שדות המחלקה)
| שם | סוג | ייעוד | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `lvLinks` | `ListView` | רכיב הממשק המציג את רשימת הקישורים. | מאותחל ב-`onCreateView`. |
| `all_links` | `ArrayList<Link>` | אוסף של אובייקטי `Link` המכילים את כל הפרטים. | בשימוש ב-`buildLinks` ובלחיצה על פריט. |
| `links_names` | `ArrayList<String>` | רשימה של שמות הקישורים בלבד, עבור המתאם (Adapter). | למילוי ה-`ListView`. |
| `internetConnectionReceiver` | `InternetConnectionReceiver` | מנטר את מצב החיבור לאינטרנט בזמן אמת. | נרשם ב-`onStart`, משוחרר ב-`onStop`. |

________________________________________
## 3. שיטות המחלקה
### שם השיטה: onCreateView
•	**מה היא עושה:**  
1. טוענת את עיצוב הפרגמנט ומוצאת את ה-`lvLinks`.  
2. קוראת ל-`buildLinks()` כדי להביא נתונים מבסיס הנתונים.  
3. מאתחלת את ה-`InternetConnectionReceiver` ומקשרת אותו לרשימה (מנטרל את הרשימה אם אין אינטרנט).  
4. מגדירה מאזין ללחיצה (`OnItemClickListener`): בהתאם לסוג הפריט (וידאו או אתר), היא קוראת לשיטה המתאימה ב-`CustomAlertDialog`.

### שם השיטה: buildLinks
•	**סוג:** private  
•	**מה היא עושה:**  
1. פותחת את בסיס הנתונים באמצעות `HelperDB`.  
2. מבצעת שאילתה לטבלת ה-`Links`.  
3. עוברת על התוצאות, יוצרת אובייקטי `Link` ומוסיפה אותם לרשימה `all_links`.  
4. מגדירה `ArrayAdapter` כדי להציג את השמות בתוך ה-`ListView`.

________________________________________
## 4. מחזור חיים
•	**onStart()**: רושם את ה-`internetConnectionReceiver` כדי להתחיל להאזין לשינויי חיבור כאשר הפרגמנט הופך לנראה.  
•	**onStop()**: מבטל את רישום המקלט כדי לחסוך במשאבי מערכת ובסוללה.

________________________________________
## 5. אינטראקציה עם הממשק (UI)
•	**ListView**: הדרך העיקרית של המשתמשים ליצור אינטראקציה עם הקישורים. משתמש בעיצוב רשימה סטנדרטי של אנדרואיד.

________________________________________
## 6. אינטראקציה עם רכיבים אחרים
•	**SQLite**: קורא את טבלת ה-`Links` כדי למלא את המסך בנתונים.  
•	**BroadcastReceiver**: בודק זמינות אינטרנט באופן דינמי.  
•	**CustomAlertDialog**: מחלקת עזר המטפלת במשימה המורכבת של הצגת סרטוני יוטיוב או דפי אינטרנט.

________________________________________
## 7. לוגיקה כללית של המחלקה
כאשר המסך נפתח, הוא קורא את כל הקישורים השמורים מבסיס הנתונים ומציג אותם כרשימה פשוטה. אם לוחצים על פריט בזמן שיש חיבור לאינטרנט, מופיע חלון קופץ המציג את הסרטון או האתר הספציפי.

________________________________________
## 8. הסבר מופשט
**הסבר במילים פשוטות:**  
`LinksFragment` הוא כמו **קטלוג בספרייה**. הוא מפרט את כל הסרטים והספרים הזמינים. אתם יכולים לעיין בכותרות, אבל אם אתם רוצים ממש לצפות בסרט (לפתוח קישור), אתם צריכים מפתח (חיבור לאינטרנט). אם אין לכם מפתח, הקטלוג נעול ואתם יכולים רק להסתכל על הכותרות.
