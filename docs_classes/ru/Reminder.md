# Подробное описание класса: Reminder

________________________________________
## 1. Общая информация
•	**Имя класса:** Reminder  
•	**Тип:** Activity (Экран настроек)  
•	**Назначение:** Этот класс позволяет пользователю запланировать уведомление. Можно выбрать точное время через `TimePicker`, установить отложенный запуск через несколько секунд или настроить повторяющееся уведомление.  
•	**Взаимодействие:** Использует системный сервис `AlarmManager` для планирования задач и `NotificationReceiver` для их выполнения.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `etInterval` | `EditText` | Поле для ввода задержки в секундах. | В `setNotification`. |
| `timePicker` | `TimePicker` | Виджет выбора времени (часы и минуты). | Для настройки календаря. |
| `rgNotificationType`| `RadioGroup` | Группа кнопок выбора типа (одиночное/повтор). | Для изменения видимости полей. |
| `etRepeatPeriod` | `EditText` | Период повторения в секундах. | При расчете повторяющегося будильника. |

________________________________________
## 3. Методы класса
### Имя метода: onCreate
•	**Что делает:** Инициализирует элементы интерфейса. Устанавливает слушатель на `RadioGroup`: если выбран пункт "Повторяющееся", поле ввода периода становится видимым (`VISIBLE`), иначе оно скрывается (`GONE`).

### Имя метода: setNotification
•	**Тип:** private  
•	**Что делает:**  
1. Считывает данные из полей ввода (использует `try-catch`, чтобы программа не вылетела, если пользователь ввел не число).  
2. Создает объект `Calendar` и устанавливает в него время из `TimePicker`.  
3. Если задан интервал, добавляет его к текущему времени.  
4. Создает `Intent` для `NotificationReceiver` и упаковывает его в `PendingIntent` (отложенное намерение).  
5. Обращается к `AlarmManager`. Если уведомление разовое — вызывает `setExact`, если повторяющееся — `setRepeating`.  
•	**Когда вызывается:** При нажатии на кнопку "Set Notification".  
•	**Важно понимать:** Начиная с Android 12, для `setExact` требуются специальные разрешения. Если уведомление не сработало, проверьте настройки разрешений в манифесте.

________________________________________
## 4. Жизненный цикл
•	**onCreate()**: Настройка UI и логики выбора типа уведомления.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
•	**TimePicker**: Удобный графический выбор времени.  
•	**RadioButton**: Переключение между режимами работы приложения.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**AlarmManager**: Системная служба, которая может разбудить приложение в нужное время.  
•	**PendingIntent**: "Доверенность", которую приложение дает системе, чтобы та выполнила действие от его имени позже.

________________________________________
## 7. Общая логика класса
Пользователь выбирает время (например, 14:00) и тип напоминания. Приложение вычисляет время в миллисекундах и отдает приказ системе: "В это время отправь сигнал в мой `NotificationReceiver`". После этого приложение можно закрыть — система сама вспомнит о задаче.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`Reminder` — это **секретарь с будильником**. Вы говорите ему: "Разбуди меня в 8 утра" или "Напоминай мне пить воду каждые 2 часа". Секретарь записывает это в свой ежедневник (`AlarmManager`) и ставит будильник. Даже если вы уснете (закроете приложение), будильник сработает, и секретарь покажет вам записку (уведомление).
