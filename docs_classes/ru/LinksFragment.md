# Подробное описание класса: LinksFragment

________________________________________
## 1. Общая информация
•	**Имя класса:** LinksFragment  
•	**Тип:** Fragment  
•	**Назначение:** Отображает список полезных ссылок, хранящихся в базе данных. Позволяет открывать видео во встроенном плеере или сайты в WebView.  
•	**Взаимодействие:** Загружает данные через `HelperDB`, использует `InternetConnectionReceiver` для проверки сети и `CustomAlertDialog` для показа контента.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `lvLinks` | `ListView` | Список для отображения ссылок на экране. | В `onCreateView`. |
| `all_links` | `ArrayList<Link>` | Список объектов `Link` с полными данными. | В `buildLinks` и при клике. |
| `links_names` | `ArrayList<String>` | Список только названий для адаптера. | Для заполнения `ListView`. |
| `internetConnectionReceiver` | `InternetConnectionReceiver` | Приемник для отслеживания интернета. | В `onStart`/`onStop`. |

________________________________________
## 3. Методы класса
### Имя метода: onCreateView
•	**Что делает:**  
1. Инициализирует список `lvLinks`.  
2. Вызывает `buildLinks()` для загрузки данных.  
3. Создает `InternetConnectionReceiver` и привязывает его к списку (если интернета нет, список станет неактивным).  
4. Устанавливает слушатель клика на элементы списка: если тип "video" — вызывает `showDialog`, если "web" — `showWeb`.

### Имя метода: buildLinks
•	**Тип:** private  
•	**Что делает:**  
1. Открывает БД через `HelperDB`.  
2. Делает запрос (`query`) к таблице `Links`.  
3. В цикле проходит по результатам, создает объекты `Link` и добавляет их в список `all_links`.  
4. Создает `ArrayAdapter` и подключает его к `lvLinks`, чтобы названия появились на экране.

________________________________________
## 4. Жизненный цикл
•	**onStart()**: Регистрирует `internetConnectionReceiver`, чтобы начать следить за сетью.  
•	**onStop()**: Отключает ресивер, чтобы не тратить заряд батареи, когда экран не виден.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
•	**ListView**: Главный элемент для выбора ссылок. Используется стандартный `simple_list_item_1`.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**SQLite**: Чтение таблицы `Links`.  
•	**BroadcastReceiver**: Динамическая проверка интернета.  
•	**CustomAlertDialog**: Вспомогательный класс для отображения медиа-контента.

________________________________________
## 7. Общая логика класса
При открытии фрагмент считывает все сохраненные ссылки из "памяти" приложения (БД) и выводит их названия списком. Если пользователь нажимает на строку и интернет включен, открывается всплывающее окно с видео или сайтом.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`LinksFragment` — это **интерактивное меню** в ресторане. Официант (фрагмент) приносит вам меню (список), где написаны названия блюд. Но чтобы заказать и съесть блюдо (открыть видео), нужно, чтобы на кухне был свет (интернет). Если света нет, официант говорит, что заказы не принимаются.
