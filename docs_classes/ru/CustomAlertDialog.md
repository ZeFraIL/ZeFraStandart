# Подробное описание класса: CustomAlertDialog

________________________________________
## 1. Общая информация
•	**Имя класса:** CustomAlertDialog  
•	**Тип:** Обычный класс (UI Helper)  
•	**Назначение:** Этот класс отвечает за отображение красивых всплывающих окон (диалогов) с мультимедийным контентом. Он может показывать видео с YouTube или веб-страницы прямо внутри приложения.  
•	**Взаимодействие:** Вызывается из `LinksFragment` при нажатии на элемент списка. Использует внешнюю библиотеку для работы с YouTube.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | Контекст для создания диалогов. | Во всех методах показа диалогов. |

________________________________________
## 3. Методы класса
### Имя метода: showDialog (YouTube видео)
•	**Тип:** public  
•	**Параметры:** `String videoId` (уникальный код видео на YouTube).  
•	**Что делает:**  
1. Создает `AlertDialog.Builder`.  
2. Загружает специальную разметку из `dialog_layout.xml`.  
3. Находит в ней `YouTubePlayerView`.  
4. Подключает жизненный цикл Activity к плееру (чтобы видео вставало на паузу при сворачивании).  
5. Добавляет слушатель, который автоматически включает видео по переданному ID.  
6. Добавляет кнопку "Close" для закрытия.

### Имя метода: showWeb (Web-страница)
•	**Тип:** public  
•	**Параметры:** `String siteAddress`  
•	**Что делает:**  
1. Программно (без XML) создает элемент `WebView`.  
2. Настраивает его: включает поддержку JavaScript и устанавливает клиентов для обработки ссылок.  
3. Создает вертикальный `LinearLayout`, в который помещает `WebView` и кнопку "Close".  
4. Показывает всё это внутри `AlertDialog`.  
•	**Важно понимать:** Здесь используется динамическое создание интерфейса через код, что позволяет не создавать много XML-файлов.

________________________________________
## 4. Жизненный цикл
Не применимо. Однако метод `showDialog` связывается с жизненным циклом вызывающей Activity через `getLifecycle().addObserver()`.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
•	**YouTubePlayerView**: Специальный виджет для воспроизведения видео.  
•	**WebView**: Браузер внутри приложения.  
•	**AlertDialog**: Основа всплывающего окна.

________________________________________
## 6. Взаимодействие с другими компонентами
Использует внешнюю библиотеку `com.pierfrancescosoffritti.androidyoutubeplayer`.

________________________________________
## 7. Общая логика класса
Этот класс — "экранный мастер". Когда ему дают команду "покажи видео", он быстро собирает окно из нужных деталей (плеер, кнопки) и выводит его на передний план.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`CustomAlertDialog` — это как **передвижной кинотеатр** или **планшет**, который вам приносят. Вам не нужно уходить со своего места (из списка ссылок), чтобы посмотреть ролик. Прямо перед вами открывается небольшое окно, в котором проигрывается видео или открывается сайт, а когда вы закончите — вы просто закрываете его и оказываетесь там же, где были.
