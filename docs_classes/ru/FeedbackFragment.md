# Подробное описание класса: FeedbackFragment

________________________________________
## 1. Общая информация
•	**Имя класса:** FeedbackFragment  
•	**Тип:** Fragment  
•	**Назначение:** Предоставляет пользователю возможность оставить отзыв тремя способами: напечатать текст, продиктовать его голосом (Speech-to-Text) или записать аудиосообщение.  
•	**Взаимодействие:** Использует системные службы распознавания речи и записи звука. Сохраняет текстовые отзывы в файл `feedback.txt`.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `etFeedback` | `EditText` | Поле для ввода или отображения распознанного текста. | В методах сохранения и STT. |
| `speechLauncher` | `ActivityResultLauncher` | Объект для запуска системного окна распознавания речи. | В методе `startSpeechToText`. |
| `mediaRecorder` | `MediaRecorder` | Инструмент для записи звука с микрофона. | В методах записи аудио. |
| `isInternetAvailable` | `boolean` | Флаг наличия интернета (нужен для работы STT). | В `updateUIState`. |

________________________________________
## 3. Методы класса
### Имя метода: onCreateView
•	**Что делает:** Настраивает кнопки. Кнопка "Записать аудио" меняет текст ("Start/Stop") в зависимости от состояния. Кнопка "Голосовой ввод" проверяет наличие интернета.

### Имя метода: startSpeechToText
•	**Тип:** private  
•	**Что делает:** Создает специальный `Intent` (намерение) для вызова системного распознавателя речи. Указывает язык (системный по умолчанию) и запускает его через `speechLauncher`.

### Имя метода: saveTextToFile
•	**Тип:** private  
•	**Параметры:** `String text`  
•	**Что делает:** Открывает файл `feedback.txt` во внутреннем хранилище приложения и дописывает туда переданный текст. Использует блок `try-with-resources` для автоматического закрытия файла.

### Имя метода: startAudioRecording / stopAudioRecording
•	**Что делает:** `start` инициализирует `MediaRecorder`, устанавливает источник звука (микрофон) и формат файла (`.3gp`), после чего начинает запись. `stop` останавливает запись и освобождает ресурсы микрофона.

________________________________________
## 4. Жизненный цикл
•	**onResume()**: Регистрирует `internetReceiver`, чтобы кнопка голосового ввода вовремя блокировалась/разблокировалась.  
•	**onPause()**: Отключает ресивер.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
•	**CheckBox (chbSayWrite)**: Условие для активации голосового ввода.  
•	**SimpleTextWatcher**: Слушатель текста, который делает кнопку "Сохранить" активной только если в поле что-то написано.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**RecognizerIntent**: Стандартный компонент Android для распознавания речи.  
•	**MediaRecorder**: Системный сервис для захвата аудио.  
•	**Internal Storage**: Место хранения файлов отзывов.

________________________________________
## 7. Общая логика класса
Пользователь может написать отзыв руками или нажать кнопку и продиктовать его. Продиктованный текст появится в поле ввода, после чего его можно сохранить в текстовый файл. Также можно просто записать свой голос в аудиофайл.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`FeedbackFragment` — это **умный почтовый ящик**. Вы можете положить в него обычное письмо (напечатать текст), надиктовать письмо секретарю, который его запишет (голосовой ввод), или оставить сообщение на автоответчик (запись аудио). Все ваши письма бережно складываются в специальную папку (файл) внутри приложения.
