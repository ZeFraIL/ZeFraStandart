# Подробное описание класса: BaseMenuClass

________________________________________
## 1. Общая информация
•	**Имя класса:** BaseMenuClass  
•	**Тип:** Обычный вспомогательный класс (Utility Class)  
•	**Назначение:** Этот класс отвечает за централизованное управление выпадающим меню (PopupMenu) и синтезом речи (Text-to-Speech) во всем приложении. Он позволяет не дублировать код обработки меню в каждой Activity.  
•	**Взаимодействие:** Вызывается из `Welcome` и других экранов. Взаимодействует с `My_Toast` для вывода уведомлений и `Reminder` для перехода по разделам.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | Контекст для работы с UI и ресурсами. | Во всех методах класса. |
| `textToSpeech` | `TextToSpeech` | Объект для преобразования текста в голос. | В `initTextToSpeech` и `speak`. |
| `isTextToSpeechEnabled` | `boolean` | Флаг включения/выключения озвучки. | В методе `speak`. |
| `isChargerCheckEnabled` | `boolean` | Состояние чекбокса озвучки в меню. | В `showPopupMenu`. |

*Примечание: Переменная `context` помечена как `static`, что может привести к утечкам памяти, если не обнулять её при закрытии приложения.*

________________________________________
## 3. Методы класса
### Имя метода: showPopupMenu
•	**Тип:** public  
•	**Возвращаемое значение:** void  
•	**Параметры:** `View anchorView` (элемент, у которого появится меню), `int menuResId` (ID файла меню).  
•	**Что делает:** Создает и показывает `PopupMenu`. Настраивает состояние пунктов меню (например, галочку у пункта TTS) и устанавливает слушатель кликов.

### Имя метода: handleMenuItemClick
•	**Тип:** private  
•	**Что делает:** Содержит всю логику кликов по меню:
1. Переключает "галочки" (checked) у пунктов.
2. `action_tts_check`: Включает/выключает звук.
3. `action_reminder`: Открывает раздел напоминаний.
4. `action_go_back`: Закрывает текущий экран.
5. `action_exit`: Закрывает всё приложение (`finishAffinity`).
6. `action_restart`: Перезагружает текущий экран.

### Имя метода: initTextToSpeech
•	**Тип:** private  
•	**Что делает:** Инициализирует движок синтеза речи и устанавливает английский язык. Выводит уведомление об успехе или ошибке.

### Имя метода: speak
•	**Тип:** public static  
•	**Параметры:** `String text`  
•	**Что делает:** Если озвучка включена, произносит переданный текст голосом.
•	**Важно понимать:** Это статический метод, его можно вызвать из любого места программы: `BaseMenuClass.speak("Привет")`.

________________________________________
## 4. Жизненный цикл
Не применимо, но есть метод `releaseTextToSpeech()`, который нужно вызывать в `onDestroy` Activity, чтобы остановить голос.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
•	**PopupMenu**: Главный элемент управления.  
•	**My_Toast**: Используется для отображения красивых всплывающих уведомлений.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**Intent**: Для перехода к `Reminder.class`.  
•	**Activity**: Использует методы `finish()`, `finishAffinity()` и `recreate()` для управления состоянием экранов.

________________________________________
## 7. Общая логика класса
Класс создается один раз в Activity. При создании он сразу готовит "голос". Когда пользователь нажимает на иконку меню, этот класс рисует список опций. Если пользователь включает звук, класс начинает "говорить" через метод `speak`.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
Представьте, что `BaseMenuClass` — это **швейцарский нож** для вашего приложения. В нем собраны инструменты, которые нужны на каждом экране: кнопка "Назад", кнопка "Выход" и даже "говорящий попугай" (синтезатор речи). Вместо того чтобы носить все эти инструменты по отдельности в каждом кармане (классе), мы просто берем этот "нож" и пользуемся им там, где нужно.

**Совет по улучшению:** Переменную `context` лучше сделать не статической, а передавать её в методы, либо использовать `WeakReference`, чтобы избежать проблем с памятью в больших приложениях.
