# Подробное описание класса: HelperDB

________________________________________
## 1. Общая информация
•	**Имя класса:** HelperDB  
•	**Тип:** SQLiteOpenHelper (Класс-помощник для работы с базой данных)  
•	**Назначение:** Этот класс отвечает за создание, управление и обновление локальной базы данных SQLite в приложении. Он определяет структуру таблиц для пользователей и ссылок.  
•	**Взаимодействие:** Используется любым компонентом приложения, которому нужно сохранить или прочитать данные из БД (например, `SignUpFragment`, `LinksFragment`).

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `FILE_DB_NAME` | `String` | Имя файла базы данных (`info.db`). | В конструкторе. |
| `TABLE_USER` | `String` | Название таблицы пользователей. | В `onCreate` и запросах. |
| `USER_NAME` | `String` | Имя столбца для логина. | В SQL-запросах. |
| `LINKS_TABLE` | `String` | Название таблицы ссылок. | В `onCreate` и запросах. |
| `LINK_ID` | `String` | Имя столбца для ID ссылки. | В SQL-запросах. |

*Примечание: Все переменные объявлены как `public static final`, что делает их константами, доступными из любого места программы без создания объекта класса.*

________________________________________
## 3. Методы класса
### Конструктор HelperDB
•	**Тип:** public  
•	**Параметры:** `Context context`  
•	**Что делает:** Вызывает конструктор суперкласса, передавая имя БД и версию (1). Это подготавливает систему к работе с файлом базы данных.

### Имя метода: onCreate
•	**Тип:** public  
•	**Возвращаемое значение:** void  
•	**Параметры:** `SQLiteDatabase db` (объект базы данных).  
•	**Что делает:** Выполняет SQL-команды `CREATE TABLE` для создания двух таблиц: `Users` (для хранения данных пользователей) и `Links` (для хранения информации о ссылках).  
•	**Когда вызывается:** Один раз, когда приложение впервые пытается обратиться к базе данных и обнаруживает, что файла БД еще не существует.

### Имя метода: onUpgrade
•	**Тип:** public  
•	**Параметры:** `db`, `oldVersion`, `newVersion`.  
•	**Что делает:** В текущем коде метод пуст. Предназначен для обновления структуры таблиц при изменении версии БД.

________________________________________
## 4. Жизненный цикл
Не является Activity, но управляется системой:  
- `onCreate()` вызывается при первом создании файла.  
- `onOpen()` (не переопределен) вызывается при каждом открытии.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
Напрямую с UI не взаимодействует. Результаты его работы отображаются в UI через `ListView` или другие элементы в других классах.

________________________________________
## 6. Взаимодействие с другими компонентами
Любой класс может создать экземпляр `HelperDB(context)` и вызвать `getWritableDatabase()` для записи или `getReadableDatabase()` для чтения.

________________________________________
## 7. Общая логика класса
Класс служит "архитектором" и "библиотекарем". Он знает, какие "полки" (таблицы) должны быть в "шкафу" (базе данных) и как они называются. Когда программа просит доступ к данным, `HelperDB` открывает нужный файл.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
Представьте, что `HelperDB` — это **заведующий складом**. Когда вы впервые приходите на пустой склад, он строит там стеллажи (`Users` и `Links`). В дальнейшем, когда вам нужно что-то положить или забрать, вы обращаетесь к нему. Он гарантирует, что все данные будут лежать на своих местах в файле `info.db`.

**Совет по улучшению:** В методе `onUpgrade` принято удалять старые таблицы и создавать новые, либо изменять существующие, чтобы приложение не "падало" при обновлении версии базы данных.
