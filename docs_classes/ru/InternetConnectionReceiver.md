# Подробное описание класса: InternetConnectionReceiver

________________________________________
## 1. Общая информация
•	**Имя класса:** InternetConnectionReceiver  
•	**Тип:** BroadcastReceiver (Приемник системных событий)  
•	**Назначение:** Этот класс "слушает" изменения в состоянии интернет-подключения устройства. Если интернет пропадает, он блокирует элементы интерфейса (например, список ссылок), чтобы пользователь не мог на них нажать.  
•	**Взаимодействие:** Регистрируется в `LinksFragment`. Взаимодействует с любым объектом `View`, который ему передали для управления.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `targetView` | `View` | Элемент интерфейса, который нужно заблокировать/разблокировать. | В методе `onReceive`. |

________________________________________
## 3. Методы класса
### Конструктор InternetConnectionReceiver
•	**Параметры:** `View targetView`  
•	**Что делает:** Сохраняет ссылку на элемент (например, `ListView`), которым нужно управлять в зависимости от наличия сети.

### Имя метода: onReceive
•	**Тип:** public  
•	**Параметры:** `Context context`, `Intent intent`  
•	**Что делает:**  
1. Получает доступ к системной службе `ConnectivityManager`.  
2. Проверяет текущие возможности сети (`NetworkCapabilities`).  
3. Определяет, подключен ли Wi-Fi или мобильный интернет.  
4. Вызывает `targetView.setEnabled(isInternetAvailable)`. Это делает элемент кликабельным или "серым" (неактивным).  
5. Если интернета нет, показывает красное уведомление через `My_Toast`.  
•	**Когда вызывается:** Автоматически системой Android при любом изменении состояния сети (включили авиарежим, выключили Wi-Fi и т.д.).

________________________________________
## 4. Жизненный цикл
Не применимо. Ресивер работает только тогда, когда он зарегистрирован в Activity или Fragment.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
Напрямую меняет состояние `Enabled` у переданного ему элемента `View`.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**ConnectivityManager**: Системный сервис для опроса состояния сетевых адаптеров.

________________________________________
## 7. Общая логика класса
Это "сторож", который постоянно проверяет, работает ли интернет. Как только "связь обрывается", он вешает замок на указанную дверь (элемент UI). Как только связь восстанавливается — он снимает замок.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`InternetConnectionReceiver` — это как **датчик протечки воды**. Он не подает воду сам, но если он "чувствует", что что-то не так (интернета нет), он перекрывает кран (блокирует кнопку). Это нужно для того, чтобы приложение не пыталось загрузить видео, когда это всё равно невозможно, и не выдавало ошибок.
