# Подробное описание класса: LoginFragment

________________________________________
## 1. Общая информация
•	**Имя класса:** LoginFragment  
•	**Тип:** Fragment (Модульный экран)  
•	**Назначение:** Этот фрагмент отвечает за экран входа пользователя в систему. Он проверяет введенные имя и пароль по базе данных.  
•	**Взаимодействие:** Находится внутри `Welcome` Activity. При успешном входе запускает `Junction` Activity и передает туда объект `User`.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `etNameL`, `etPassL` | `EditText` | Поля ввода для имени и пароля. | В `onCreateView` для получения данных. |
| `bLogin` | `Button` | Кнопка подтверждения входа. | В `onCreateView` для установки слушателя. |
| `helperDB` | `HelperDB` | Помощник для работы с SQLite. | В логике поиска пользователя. |
| `user` | `User` | Объект для хранения данных найденного пользователя. | Передается в следующее Activity. |

________________________________________
## 3. Методы класса
### Имя метода: onCreateView
•	**Тип:** public  
•	**Возвращаемое значение:** View (разметка фрагмента)  
•	**Параметры:** `LayoutInflater`, `ViewGroup`, `Bundle`  
•	**Что делает:**  
1. Создает визуальный интерфейс из файла `fragment_login`.  
2. Находит элементы управления.  
3. При нажатии на кнопку `bLogin` считывает текст из полей, проверяет их на пустоту и вызывает `isUserFound`.  
4. Если пользователь найден, создает `Intent` для перехода в `Junction`, вкладывает туда объект `user` и запускает новое окно.

### Имя метода: isUserFound
•	**Тип:** private  
•	**Возвращаемое значение:** boolean (true — если найден, false — нет)  
•	**Параметры:** `String stName`, `String stPassword`  
•	**Что делает:**  
1. Открывает базу данных для чтения.  
2. Делает запрос ко всей таблице `Users`.  
3. В цикле `while` проходит по всем строкам.  
4. Сравнивает имя или пароль из БД с введенными данными.  
5. Если совпадение найдено, создает объект `User` и возвращает `true`.

________________________________________
## 4. Жизненный цикл
•	**onCreateView()**: Вызывается для создания интерфейса. Здесь настраивается вся логика кнопок.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
•	**EditText**: Используются методы `getText().toString()` для получения данных.  
•	**My_Toast**: Выводит сообщения об ошибках (красным цветом).  
•	**BaseMenuClass.speak()**: Озвучивает ошибки входа.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**SQLite**: Прямое обращение к `HelperDB` для поиска в таблице.  
•	**Intent**: Переход к `Junction` с использованием `putExtra("user", user)`.

________________________________________
## 7. Общая логика класса
Пользователь вводит данные. Фрагмент "идет" в базу данных и ищет там совпадения. Если находит — пропускает пользователя дальше, если нет — ругается красным сообщением и голосом.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`LoginFragment` — это как **охранник на проходной**. Вы даете ему свой паспорт (имя и пароль). Он открывает толстую книгу (базу данных) и ищет ваше имя в списке. Если нашел — открывает дверь в здание (`Junction`). Если не нашел — говорит "Вас нет в списках!".

**Совет по улучшению:** В методе `isUserFound` поиск идет по всей таблице. Если пользователей будет много, это будет медленно. Лучше делать запрос сразу с условием `WHERE User_Name = ? AND User_Password = ?`.
