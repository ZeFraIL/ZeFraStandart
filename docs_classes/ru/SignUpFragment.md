# Подробное описание класса: SignUpFragment

________________________________________
## 1. Общая информация
•	**Имя класса:** SignUpFragment  
•	**Тип:** Fragment  
•	**Назначение:** Этот фрагмент отвечает за регистрацию новых пользователей. Он собирает данные (имя, пароль, почта, телефон), проверяет их и сохраняет в базу данных.  
•	**Взаимодействие:** Работает внутри `Welcome` Activity. После успешной регистрации переходит в `Junction` Activity. Использует фоновые потоки для записи данных.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `etNameSU` и др. | `EditText` | Поля для ввода данных регистрации. | В `onCreateView` для сбора текста. |
| `helperDB` | `HelperDB` | Объект для управления базой данных. | В методах `saveUser` и `isUserFound`. |
| `user` | `User` | Объект для хранения данных нового пользователя. | Для передачи данных в `Junction`. |

________________________________________
## 3. Методы класса
### Имя метода: onCreateView
•	**Что делает:**  
1. Инициализирует поля ввода и кнопку регистрации.  
2. При клике на кнопку проверяет каждое поле на пустоту. Если поле пустое — выводит предупреждение.  
3. Вызывает `isUserFound`. Если пользователя с такими данными нет — вызывает `saveUser` и переходит на следующий экран.  

### Имя метода: saveUser
•	**Тип:** private  
•	**Параметры:** `User user`  
•	**Что делает:**  
1. Создает **фоновый поток** через `ExecutorService`. Это важно, чтобы запись в базу данных не "подвешивала" интерфейс.  
2. Использует `ContentValues` для упаковки данных.  
3. Вызывает `db.insert()` для сохранения в таблицу `Users`.  
4. Возвращается в главный поток (`runOnUiThread`), чтобы показать сообщение об успехе.  
•	**Важно понимать:** Операции с БД могут быть долгими, поэтому их *обязательно* нужно делать вне главного потока.

### Имя метода: isUserFound
•	**Тип:** private  
•	**Что делает:** Ищет в БД, не занято ли уже выбранное имя или пароль. Логика аналогична `LoginFragment`.

•	**SQLite**: Использование `getWritableDatabase()` для записи новых данных.  
•	**ExecutorService**: Использование многопоточности для оптимизации работы.  
•	**Intent**: Переход в `Junction`.

________________________________________
## 7. Общая логика класса
Пользователь заполняет анкету. Приложение проверяет, нет ли уже такого пользователя в "книге". Если всё чисто — приложение в отдельном потоке записывает данные на "склад" (БД) и поздравляет пользователя переходом на главный экран.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`SignUpFragment` — это **отдел кадров**. Когда вы приходите устраиваться, вы заполняете анкету. Клерк проверяет, не работали ли вы у нас раньше. Если вы новый человек, он отправляет помощника (`executor`) отнести ваши документы в архив (базу данных). Пока помощник бегает в архив, клерк может продолжать общаться с вами, и вы не будете ждать, пока захлопнется дверь архива.
