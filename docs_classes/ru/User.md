# Подробное описание класса: User

________________________________________
## 1. Общая информация
•	**Имя класса:** User  
•	**Тип:** Обычный класс (POJO - Plain Old Java Object)  
•	**Назначение:** Этот класс представляет собой модель данных для пользователя. Он используется для хранения информации об одном пользователе: имя, пароль, телефон и электронная почта.  
•	**Взаимодействие:** Используется фрагментами `SignUpFragment` и `LoginFragment` для создания объектов пользователей и их передачи, а также может быть использован при работе с базой данных. Реализует интерфейс `Serializable`, что позволяет передавать объекты этого класса между экранами (Activity/Fragment) через `Intent` или `Bundle`.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `userName` | `String` | Логин или имя пользователя. | Во всех методах доступа (get/set). |
| `userPassword` | `String` | Пароль пользователя. | Для проверки при входе. |
| `userPhone` | `String` | Номер телефона пользователя. | Для связи или профиля. |
| `userMail` | `String` | Электронная почта пользователя. | Для уведомлений или профиля. |

________________________________________
## 3. Методы класса
### Конструктор User
•	**Тип:** public  
•	**Параметры:** `userName`, `userPassword`, `userPhone`, `userMail` (все `String`).  
•	**Что делает:** Создает новый объект пользователя и заполняет его данными.

### Геттеры и Сеттеры (например, `getUserName`, `setUserName`)
•	**Тип:** public  
•	**Что делают:** Позволяют безопасно получать или изменять значения приватных переменных класса. Это принцип **инкапсуляции**.

### Метод toString()
•	**Тип:** public  
•	**Возвращает:** `String`  
•	**Что делает:** Формирует текстовое представление объекта. Удобно для отладки, чтобы быстро увидеть все данные пользователя в консоли (Logcat).

________________________________________
## 4. Жизненный цикл
Не применимо, так как это обычный класс данных.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
Напрямую с интерфейсом не взаимодействует. Данные из полей ввода (EditText) передаются в этот класс через посредников (Fragment/Activity).

________________________________________
## 6. Взаимодействие с другими компонентами
•	**Serializable**: Позволяет "сериализовать" объект (превратить в поток байтов) для передачи между компонентами Android.

________________________________________
## 7. Общая логика класса
Это просто контейнер для данных. Мы создаем объект `User`, когда пользователь регистрируется, и используем его для хранения информации, пока приложение работает.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
Представьте, что `User` — это **анкета** в бумажной папке. В анкете есть графы: "Имя", "Пароль", "Телефон". Этот класс — просто шаблон такой анкеты. Когда мы заполняем её данными конкретного человека, мы получаем "объект". Папка (`Serializable`) позволяет нам легко переносить эту анкету из одного кабинета (экрана) в другой.
