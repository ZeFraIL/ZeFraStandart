# Подробное описание класса: Welcome

________________________________________
## 1. Общая информация
•	**Имя класса:** Welcome  
•	**Тип:** Activity (Главный экран после входа)  
•	**Назначение:** Этот класс является основным контейнером для фрагментов после запуска приложения. Он управляет нижней навигацией (Bottom Navigation) и выпадающим меню.  
•	**Взаимодействие:** Переключает фрагменты (`LoginFragment`, `SignUpFragment`, `GuestFragment`) внутри `fragment_container`. Использует `BaseMenuClass` для отображения меню.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | Контекст приложения. | Для инициализации меню. |
| `ivPUM` | `ImageView` | Иконка для вызова PopupMenu. | В `onCreate` для установки клика. |
| `baseMenuClass` | `BaseMenuClass` | Объект вспомогательного класса для меню. | Для вызова методов меню и очистки ресурсов. |

________________________________________
## 3. Методы класса
### Имя метода: onCreate
•	**Тип:** protected  
•	**Возвращаемое значение:** void  
•	**Параметры:** `Bundle savedInstanceState`  
•	**Что делает метод:**  
1. Устанавливает разметку `activity_welcome`.  
2. Инициализирует иконку меню (`ivPUM`) и `baseMenuClass`.  
3. Настраивает `BottomNavigationView`: при выборе элемента меню заменяет текущий фрагмент в контейнере на выбранный (`Login`, `SignUp` или `Guest`).  
4. При первом запуске (`savedInstanceState == null`) автоматически загружает `LoginFragment`.  
•	**Когда вызывается:** При создании Activity.

### Имя метода: onDestroy
•	**Тип:** protected  
•	**Что делает:** Вызывает `releaseTextToSpeech()` у объекта `baseMenuClass`, чтобы корректно освободить память, занятую синтезатором речи.  
•	**Когда вызывается:** При закрытии приложения или уничтожении экрана.

________________________________________
## 4. Жизненный цикл
•	**onCreate()**: Инициализация навигации и загрузка первого фрагмента.  
•	**onDestroy()**: Освобождение ресурсов (TTS).

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
•	**BottomNavigationView**: Слушает выбор вкладок и переключает контент экрана.  
•	**ImageView (ivPUM)**: Работает как кнопка меню в верхней части экрана.  
•	**FragmentContainerView (id: fragment_container)**: Область, в которой меняются фрагменты.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**FragmentManager**: Используется для замены (`replace`) и управления стеком фрагментов (`addToBackStack`).  
•	**BaseMenuClass**: Делегирует управление меню и TTS отдельному классу.

________________________________________
## 7. Общая логика класса
Класс работает как "пульт управления". У него есть нижняя панель с кнопками. Когда пользователь нажимает "Вход", "Регистрация" или "Гость", `Welcome` просто меняет "картинку" (фрагмент) в центре экрана.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
Представьте, что `Welcome` — это **рамка для телевизора**. Сама рамка не меняется, но у вас есть пульт (нижнее меню). Нажимая на разные кнопки пульта, вы меняете каналы (фрагменты) на экране. Также у вас есть кнопка настроек (иконка меню) для дополнительных действий.

**Совет по улучшению:** В коде используется `if (item.getItemId() == ...)` для навигации. В современных версиях Android Studio рекомендуется использовать `switch-case`, если это возможно, или более современные библиотеки навигации (Jetpack Navigation).
