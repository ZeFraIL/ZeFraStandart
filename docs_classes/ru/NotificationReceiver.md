# Подробное описание класса: NotificationReceiver

________________________________________
## 1. Общая информация
•	**Имя класса:** NotificationReceiver  
•	**Тип:** BroadcastReceiver (Приемник широковещательных сообщений)  
•	**Назначение:** Этот класс ловит "сигнал будильника" от системы и превращает его в красивое уведомление в шторке телефона.  
•	**Взаимодействие:** Запускается системой по расписанию, установленному в `Reminder`. Вызывает `CloseNotificationService` для закрытия уведомления.

________________________________________
## 2. Переменные (Поля класса)
В этом классе нет глобальных переменных, все данные приходят внутри метода `onReceive`.

________________________________________
## 3. Методы класса
### Имя метода: onReceive
•	**Тип:** public  
•	**Параметры:** `Context context`, `Intent intent`  
•	**Что делает:**  
1. Достает заголовок и текст напоминания из `intent`.  
2. Создает `RemoteViews`. Это специальный объект для управления интерфейсом уведомления (так как уведомление рисуется системой, а не нашим приложением).  
3. Устанавливает текст и картинку в макет уведомления (`notification_layout`).  
4. Настраивает кнопки внутри уведомления:
   - Кнопка "Закрыть" запускает `CloseNotificationService`.
   - Кнопка "Открыть" возвращает пользователя в приложение (в экран `Reminder`).
5. Собирает уведомление через `NotificationCompat.Builder`, указывая канал важности (`CHANNEL_HIGH`).  
6. Отправляет уведомление в системный менеджер для показа.  
•	**Когда вызывается:** Когда срабатывает будильник в `AlarmManager`.  
•	**Важно понимать:** `RemoteViews` имеет ограниченный набор виджетов. Например, в нем нельзя использовать сложные анимации или кастомные списки.

________________________________________
## 4. Жизненный цикл
Работает мгновенно: получил сигнал -> создал уведомление -> завершил работу.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
Использует **кастомный макет уведомления**. Это позволяет сделать уведомление более красивым и функциональным (с кнопками), чем стандартное системное.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**NotificationManager**: Главный системный сервис для управления всеми уведомлениями на устройстве.  
•	**PendingIntent.getService**: Для выполнения действия в фоне без открытия экрана.  
•	**PendingIntent.getActivity**: Для открытия приложения при клике.

________________________________________
## 7. Общая логика класса
Класс работает как **курьер**. Система (будильник) говорит ему: "Пора!". Курьер берет заранее заготовленный шаблон (макет), вписывает туда нужный адрес (заголовок) и вешает на дверь пользователю (в шторку уведомлений).

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`NotificationReceiver` — это **исполнитель поручений**. Представьте, что вы оставили в календаре запись: "В 10 утра отправить письмо". Система в 10 утра "будит" этого исполнителя. Он быстро рисует красивую открытку (уведомление) с кнопками и вешает её вам перед глазами, чтобы вы точно ничего не забыли.
