# Подробное описание класса: BatteryStatusReceiver

________________________________________
## 1. Общая информация
•	**Имя класса:** BatteryStatusReceiver  
•	**Тип:** BroadcastReceiver  
•	**Назначение:** Отслеживает уровень заряда батареи устройства. Если уровень становится критически низким (меньше 15%), класс выводит специальное предупреждение на экран.  
•	**Взаимодействие:** Регистрируется динамически через `BaseMenu`. Получает системные сообщения о состоянии батареи.

________________________________________
## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `lastShownPercentage` | `int` | Хранит последний процент заряда, при котором было показано уведомление. | Чтобы не показывать сообщение каждую секунду, если заряд не изменился. |

________________________________________
## 3. Методы класса
### Имя метода: onReceive
•	**Тип:** public  
•	**Параметры:** `Context context`, `Intent intent`  
•	**Что делает:**  
1. Проверяет, что пришедшее событие — это действительно `ACTION_BATTERY_CHANGED`.  
2. Вычисляет текущий процент заряда по формуле: `(level / scale) * 100`.  
3. Проверяет условие: если заряд <= 15% и он уменьшился с момента последнего показа.  
4. Создает **программный кастомный Toast**:
   - Создает `LinearLayout` с красным фоном.
   - Добавляет туда иконку батареи (`ImageView`).
   - Добавляет текстовое сообщение с текущим процентом.
5. Выводит созданный дизайн в центр экрана на долгое время.  
•	**Когда вызывается:** Автоматически системой при любом изменении уровня заряда батареи (обычно на 1%).  
•	**Важно понимать:** В этом классе интерфейс создается полностью через Java-код (без XML), что демонстрирует гибкость Android в создании динамических уведомлений.

________________________________________
## 4. Жизненный цикл
Работает мгновенно в момент получения системного сигнала.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
Использует кастомный `Toast`. В современных версиях Android кастомные Toast-уведомления ограничены, когда приложение находится в фоне, но здесь они работают, так как привязаны к активному контексту.

________________________________________
## 6. Взаимодействие с другими компонентами
•	**BatteryManager**: Системный класс, предоставляющий константы для извлечения данных о батарее из `Intent`.

________________________________________
## 7. Общая логика класса
Класс работает как **датчик топлива**. Он постоянно следит за стрелкой бака. Как только стрелка заходит в "красную зону", он включает яркую красную лампочку на приборной панели, чтобы водитель (пользователь) не забыл подзарядить устройство.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
`BatteryStatusReceiver` — это **сигнализация низкого заряда**. Представьте, что у вас есть маленький помощник, который раз в минуту заглядывает в настройки телефона. Если он видит, что батарейки осталось совсем мало, он выбегает на середину экрана с красным плакатом и кричит: "Внимание! Срочно найди зарядку!".
