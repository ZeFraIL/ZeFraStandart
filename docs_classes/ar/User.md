# وصف مفصل للفئة: User

________________________________________
## 1. معلومات عامة
•	**اسم الفئة:** User  
•	**النوع:** فئة عادية (POJO - Plain Old Java Object)  
•	**الغرض:** تمثل هذه الفئة نموذج بيانات للمستخدم. تُستخدم لتخزين معلومات حول مستخدم واحد: الاسم، كلمة المرور، الهاتف، والبريد الإلكتروني.  
•	**التفاعل:** تُستخدم بواسطة `SignUpFragment` و `LoginFragment` لإنشاء كائنات (Objects) للمستخدمين. تقوم بتنفيذ واجهة `Serializable` مما يسمح بنقل كائناتها بين الشاشات (Activities/Fragments) عبر `Intent` أو `Bundle`.

________________________________________
## 2. المتغيرات (حقول الفئة)
| الاسم | النوع | الغرض | أين يستخدم |
| :--- | :--- | :--- | :--- |
| `userName` | `String` | اسم تسجيل دخول المستخدم. | في جميع طرق الـ get والـ set. |
| `userPassword` | `String` | كلمة مرور المستخدم. | للتحقق عند تسجيل الدخول. |
| `userPhone` | `String` | رقم هاتف المستخدم. | للتواصل أو عرض الملف الشخصي. |
| `userMail` | `String` | البريد الإلكتروني للمستخدم. | للإشعارات أو عرض الملف الشخصي. |

________________________________________
## 3. طرق الفئة (Methods)
### المنشئ (Constructor): User(...)
•	**النوع:** public  
•	**المعاملات:** `userName`, `userPassword`, `userPhone`, `userMail` (جميعها `String`).  
•	**ماذا يفعل:** ينشئ كائن مستخدم جديد ويملأ حقوله بالبيانات المزودة.

### طرق الحصول والتعيين (مثل `getUserName`, `setUserName`)
•	**النوع:** public  
•	**ماذا تفعل:** تسمح بالوصول الآمن إلى المتغيرات الخاصة (private) للفئة أو تعديلها. هذا يتبع مبدأ **التغليف** (Encapsulation).

### طريقة: toString()
•	**النوع:** public  
•	**قيمة الإرجاع:** `String`  
•	**ماذا تفعل:** ترجع تمثيلاً نصياً للكائن. هذا مفيد جداً للتصحيح (Debugging) لرؤية بيانات المستخدم في سجل النظام (Logcat).

________________________________________
## 4. دورة الحياة
لا ينطبق (هذه فئة بيانات وليست مكون واجهة مستخدم).

________________________________________
## 5. التفاعل مع الواجهة (UI)
لا تتفاعل مع الواجهة بشكل مباشر. البيانات من عناصر الإدخال (EditText) يتم جمعها بواسطة Fragment أو Activity ثم تُستخدم لتعبئة هذه الفئة.

________________________________________
## 6. التفاعل مع المكونات الأخرى
•	**Serializable**: يسمح بتحويل الكائن إلى تدفق من البايتات ليتم نقله بين الشاشات.

________________________________________
## 7. المنطق العام للفئة
تعمل الفئة كـ "حاوية" بسيطة للبيانات. نقوم بإنشاء كائن `User` عندما يسجل شخص ما، ونستخدمه لنقل معلوماته داخل التطبيق.

________________________________________
## 8. شرح مبسط
**الشرح بكلمات بسيطة:**  
تخيل أن فئة `User` هي عبارة عن **استمارة بيانات** أو بطاقة تعريف. في البطاقة توجد خانات محددة لـ "الاسم" و "كلمة المرور" و "الهاتف". الفئة نفسها هي القالب الفارغ لهذه البطاقة. عندما نكتب فيها بيانات شخص معين، نكون قد أنشأنا "كائن". أما ميزة `Serializable` فهي تشبه وضع هذه البطاقة في مظروف حتى نتمكن من إرسالها من مكتب (شاشة) إلى آخر.
