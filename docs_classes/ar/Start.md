# وصف مفصل للفئة: Start

________________________________________
## 1. معلومات عامة
•	**اسم الفئة:** Start  
•	**النوع:** Activity (شاشة تطبيق)  
•	**الغرض:** هذه الفئة مسؤولة عن شاشة الترحيب الأولية (Splash Screen). تقوم بتهيئة قاعدة البيانات، وإعداد ميزة تحويل النص إلى كلام (TTS)، وتوجيه المستخدم تلقائيًا إلى الشاشة التالية بعد 4 ثوانٍ.  
•	**التفاعل:** تقوم بتشغيل الـ Activity المسماة `Welcome` بعد انتهاء المؤقت أو عند الضغط على الزر. وتستخدم `HelperDB` لتجهيز قاعدة البيانات.

________________________________________
## 2. المتغيرات (حقول الفئة)
| الاسم | النوع | الغرض | أين يستخدم |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | يحفظ معلومات حول الحالة الحالية للتطبيق. | في تهيئة قاعدة البيانات وTTS. |
| `ivLogo` | `ImageView` | عنصر لعرض الشعار (الصورة). | في طريقة `onCreate` لعمل الحركة (Animation). |
| `bStart` | `Button` | زر للانتقال اليدوي. | في `onCreate` لمعالجة الضغط عليه. |
| `cdt` | `CountDownTimer` | مؤقت للعد التنازلي. | للانتقال التلقائي بعد 4 ثوانٍ. |
| `totalTTS` | `TextToSpeech` | متغير ثابت (static) لتحويل النص إلى كلام. | يتم تهيئته هنا ويمكن استخدامه في فئات أخرى. |

________________________________________
## 3. طرق الفئة (Methods)
### اسم الطريقة: onCreate
•	**نوع الطريقة:** protected  
•	**قيمة الإرجاع:** void (لا ترجع شيئاً)  
•	**المعاملات (Parameters):**  
| الاسم | النوع | الوصف |
| :--- | :--- | :--- |
| `savedInstanceState` | `Bundle` | حالة الشاشة عند الاستعادة. |
•	**ماذا تفعل الطريقة:**  
1. تحدد شكل الواجهة من ملف XML (`activity_start`).  
2. تهيئ قاعدة البيانات عبر `HelperDB` للتأكد من إنشائها.  
3. تهيئ `totalTTS` للسماح بنطق النصوص.  
4. تربط عناصر الواجهة (`ivLogo`, `bStart`) بالكود البرمجي.  
5. تبدأ حركة دوران للشعار بزاوية 360 درجة.  
6. تضبط مستمعًا للنقر على زر `bStart` يقوم بإلغاء المؤقت والانتقال إلى شاشة `Welcome`.  
7. تبدأ المؤقت `cdt` لمدة 4 ثوانٍ.  
•	**متى يتم استدعاؤها:** تلقائيًا بواسطة النظام عند إنشاء الشاشة.  
•	**ما المهم فهمه:** إذا تم الضغط على الزر، يجب إلغاء المؤقت (`cancel`) لمنع حدوث انتقال مزدوج.

________________________________________
## 4. دورة الحياة (Activity/Fragment فقط)
•	**onCreate()**: تُستدعى عند بدء التطبيق. هنا يتم إعداد جميع المكونات.

________________________________________
## 5. التفاعل مع الواجهة (UI)
•	**ImageView (ivLogo)**: يستخدم لعمل حركة الدوران. يتم ربطه عبر `findViewById`.  
•	**Button (bStart)**: يعالج عملية الضغط (`OnClickListener`).

________________________________________
## 6. التفاعل مع المكونات الأخرى
•	**Intent**: يستخدم للانتقال إلى `Welcome.class`.  
•	**SQLite**: يتم استدعاء `getWritableDatabase()` عبر `HelperDB` لإنشاء الجداول إذا لم تكن موجودة.

________________________________________
## 7. المنطق العام للفئة
عند تشغيل التطبيق، يرى المستخدم شعارًا يدور. في هذا الوقت، يتم تجهيز قاعدة البيانات في الخلفية. بعد 4 ثوانٍ (أو عند الضغط على الزр)، ينتقل التطبيق إلى الشاشة الرئيسية.

________________________________________
## 8. شرح مبسط
**الشرح بكلمات بسيطة:**  
تخيل أن هذه هي "ردهة الاستقبال" في منزلك. بينما يقف الضيف هنا، تقوم أنت بالتحقق من أن كل شيء جاهز في الغرف (قاعدة البيانات) وتشغل الموسيقى (الصوت). لكي لا ينتظر الضيف طويلاً، تقوم أنت بعد 4 ثوانٍ بمرافقته إلى غرفة المعيشة (`Welcome`) بنفسك، أو يمكنه الدخول إلى هناك في وقت أبكر بالضغط على زر "دخول".

**نصيحة للتحسين:** المتغير `totalTTS` تم تعريفه كـ `static`. هذا مريح ولكنه قد يؤدي إلى تسريب في الذاكرة إذا لم يتم إغلاق محرك الـ TTS بشكل صحيح.
